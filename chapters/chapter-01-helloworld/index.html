Hello World

Der Einstieg: ein Objekt auf die Bühne bringen, Text anzeigen, und verstehen, wann im Lebenszyklus einer Klasse welcher Code läuft. Original: A3GPU201_HelloWorld (HelloWorld1–3, DebugExample, TooEarlyExample).

Worum es geht

In Flash war die "Bühne" (Stage) ein Objektbaum, auf den Zeitleisten-Frames und ActionScript-Klassen gleichzeitig einwirken konnten. Im Web haben wir dafür kein Zeitleisten-Konzept mehr — stattdessen ein <canvas>-Element, auf das wir per JavaScript direkt zeichnen, und die Browser-Konsole statt des Flash-"Ausgabe"-Fensters für trace().

Lern-Idee dieses Kapitels: trace() → console.log(), und TextField auf der Bühne → Text auf dem Canvas zeichnen. Das sind die zwei Übersetzungen, die durch fast das ganze restliche Buch durchgezogen werden.

1) Nur eine Konsolen-Ausgabe

Das simpelste Beispiel im Buch schreibt nichts auf die Bühne, nur ins Ausgabefenster.

ActionScript 3 
public class HelloWorld1
  extends MovieClip {

  public function HelloWorld1() {
    trace("Hello World.");
  }
}
JavaScript 
function helloWorld1() {
  console.log("Hello World.");
}

2) Sichtbarer Text auf der Bühne

HelloWorld3 erzeugt ein TextField und hängt es per addChild() in die Anzeigeliste. Auf dem Canvas gibt es keine Anzeigeliste einzelner Objekte — wir zeichnen den Text direkt in den Frame-Buffer.

ActionScript 3 
var myText:TextField
  = new TextField();
myText.text = "Hello World!";
addChild(myText);
JavaScript 
ctx.font = '24px monospace';
ctx.fillStyle = '#ffb84d';
ctx.fillText(
  'Hello World!', 20, 20
);

3) Debug-Ausgaben mit Positionierung

DebugExample erzeugt in einer Schleife zehn Textfelder, jedes 20 Pixel tiefer als das vorige — eine simple, aber wirkungsvolle Methode, um Programmzustände sichtbar zu machen, bevor man einen echten Debugger nutzt.

ActionScript 3 
for (var i:int=0; i<10; i++) {
  showNumber(i);
}

function showNumber(n:int) {
  var t:TextField = new TextField();
  t.text = String(n);
  t.y = n * 20;
  addChild(t);
}
JavaScript 
for (let i = 0; i < 10; i++) {
  showNumber(i);
}

function showNumber(n) {
  ctx.fillText(String(n), 20, n * 22);
  console.log('showNumber:', n);
}

4) Der Klassiker: zu früh zugegriffen

TooEarlyExample zeigt einen Fehler, den praktisch jeder AS3- und JS-Einsteiger macht: Der Konstruktor eines Objekts läuft, bevor seine Kind-Objekte (dort: ein Textfeld auf Frame 2 der Zeitleiste) existieren. Im Web ist die Entsprechung: Code läuft, bevor das DOM fertig geladen ist, oder bevor ein fetch() zurückgekommen ist.

ActionScript 3 — schlägt fehl 
function TooEarlyExample() {
  gotoAndStop(2);
  // existiert erst ab Frame 2!
  this.textFieldOne.text = "...";
}
JavaScript — dieselbe Falle 
const el = document.getElementById('x');
el.textContent = '...';
// ReferenceError, wenn das Skript
// vor dem <div id="x"> im HTML steht
Lösung in beiden Welten: entweder auf das richtige Lebenszyklus-Ereignis warten (DOMContentLoaded / window.onload) oder den Code erst ausführen, nachdem das referenzierte Objekt wirklich erzeugt wurde.
