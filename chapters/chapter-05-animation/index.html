Animation

Das denkbar einfachste Bewegungsbeispiel: 50 Objekte, jedes mit eigener konstanter Geschwindigkeit. Original: A3GPU205_Animation (AnimatedObject, AnimationTest).

Jedes Objekt bewegt sich selbst

AnimatedObject ist eine eigenständige Klasse mit eigenem ENTER_FRAME-Listener — jedes der 50 Objekte aktualisiert seine Position unabhängig von allen anderen. Dasselbe Muster (Objekt kennt seine eigene Geschwindigkeit, aktualisiert sich selbst über update(dt)) taucht im ganzen restlichen Buch wieder auf, von Air Raids Flugzeugen bis zu Space Rocks' Asteroiden.

ActionScript 3 
public function moveObject(
    event:Event) {
  var timePassed:int =
    getTimer() - lastTime;
  lastTime += timePassed;
  this.x +=
    speedX*timePassed/1000;
  this.y +=
    speedY*timePassed/1000;
}
JavaScript 
update(dt) {
  // dt bereits in Sekunden
  this.x += this.speedX * dt;
  this.y += this.speedY * dt;
}

Kleiner, aber praktischer Unterschied: AS3 misst in Millisekunden und teilt am Ende durch 1000, um auf "Pixel pro Sekunde" zu kommen. Im JS-Game-Loop wird dt gleich in Sekunden berechnet ((now - lastTime) / 1000), das Teilen entfällt.

Kleine Ergänzung: Objekte tauchen wieder auf

Das Original lässt Objekte einfach von der Bühne verschwinden, wenn sie den sichtbaren Bereich verlassen — nach ein paar Sekunden wäre die Demo leer. Hier tauchen sie stattdessen am gegenüberliegenden Rand wieder auf, damit die Demo dauerhaft interessant bleibt. Eine bewusste kleine Ergänzung, keine Notwendigkeit der Portierung.

Reine Beobachtungs-Demo, keine Steuerung nötig — 50 Objekte, die unabhängig voneinander mit konstanter Geschwindigkeit driften.
