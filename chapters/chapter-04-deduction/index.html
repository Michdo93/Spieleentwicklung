Deduction

Ein Mastermind-Klon: 5 farbige Stifte erraten, 10 Versuche, nach jeder Reihe Feedback über richtige Position und richtige Farbe. Original: A3GPU204_Deduction.

Reines Logikspiel, kein Timing

Anders als Air Raid oder das Matching Game läuft hier nichts von selbst — das ganze Spiel besteht aus Klick-Reaktionen. Das ist ein guter Kontrast: kein ENTER_FRAME/requestAnimationFrame-Loop nötig, nur Event-Handler.

Der Auswertungs-Algorithmus

Der interessante Teil ist calculateProgress(): "richtige Farbe, falsche Stelle" zu zählen ist ein klassisches Zähl-Problem. Der Trick im Original — für jede Farbe zählen, wie oft sie übrig (nicht exakt getroffen) in Lösung und Rateversuch vorkommt, dann das Minimum beider Zahlen aufsummieren — funktioniert unverändert in JavaScript:

ActionScript 3 
for(i=0;i<numPegs;i++) {
  if (currentRow[i].color
      == solution[i]) {
    numCorrectSpot++;
  } else {
    solutionColorList[
      solution[i]-1]++;
    currentColorList[
      currentRow[i].color-1]++;
  }
}
for(i=0;i<numColors;i++) {
  numCorrectColor += Math.min(
    solutionColorList[i],
    currentColorList[i]);
}
JavaScript 
for (let i=0; i<numPegs; i++) {

for (let i=0; i<numColors; i++) {

UI ohne TextField/Button-Symbole

Der DONE-Button und die Farbstifte waren im Original eigene Bibliothekssymbole mit Klick-Erkennung. Auf dem Canvas gibt es keine eingebauten interaktiven Elemente — ein Klick auf die Bühne wird per Koordinaten-Test der passenden Schaltfläche zugeordnet (derselbe Rechtecktest wie in Kapitel 02, Demo 7 "Kollision").

Farbstift anklicken schaltet seine Farbe weiter (leer → 5 Farben → leer), genau wie im Original. ● zeigt "richtige Stelle", ○ zeigt "richtige Farbe, falsche Stelle".

5 Farben, 10 Versuche. Finde die geheime Kombination.
