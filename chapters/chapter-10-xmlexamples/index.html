XML Examples

Der Grundbaustein für alle drei Trivia-Spiele in diesem Kapitel: externe Frage-Daten laden. AS3 benutzt dafür XML, im Web ist JSON die naheliegende Wahl. Original: A3GPU210_XMLExamples.

URLLoader + XML → fetch() + JSON

AS3 lädt externe Daten asynchron über URLLoader, hört auf Event.COMPLETE, und parst dann mit XML(event.target.data). Im Web übernimmt fetch() beides in einem Aufruf — Netzwerk- Request und Parsing sind durch await response.json() bereits kombiniert.

ActionScript 3 
var loader = new URLLoader(
  new URLRequest(
    "xmltestdata.xml"));
loader.addEventListener(
  Event.COMPLETE, xmlLoaded);
loader.addEventListener(
  IOErrorEvent.IO_ERROR,
  xmlLoadError);

function xmlLoaded(e) {
  var xmldata = XML(e.target.data);
  trace(xmldata.item.answers
    .answer[1]);
}
JavaScript 
try {
  const response = await fetch(
    "xmltestdata.json");
  const data = await response.json();
  console.log(
    data[0].answers[1]);
} catch (err) {
  console.log(
    "Fehler:", err.message);
}

Warum JSON statt XML im Browser?

Beides würde im Browser funktionieren (DOMParser kann XML parsen), aber JSON ist die native Datenstruktur von JavaScript — kein Parsing-Schritt, kein Umweg über XMLList-Zugriffe wie item.answers.answer[0]. Die Original-Fragedaten wurden 1:1 nach JSON konvertiert, inhaltlich unverändert.

XML (Original) 
<trivia>
  <item>
    <question>
      Which planet is closest
      to the sun?
    </question>
    <answers>
      <answer>Mercury</answer>
      <answer>Venus</answer>
      ...
    </answers>
  </item>
</trivia>
JSON (portiert) 
[
  {
    "question": "Which planet
      is closest to the sun?",
    "answers": [
      "Mercury", "Venus", ...
    ]
  }
]
Diese Umstellung (XML → JSON, URLLoader → fetch()) ist die Grundlage für alle drei folgenden Trivia-Spiele in diesem Kapitel.
Achtung, Browser-Eigenheit: fetch() lädt lokale Dateien nicht, wenn die Seite direkt per file:// geöffnet wird (CORS-Sperre für lokale Requests in Chrome/Edge). Ab diesem Kapitel lokal am besten mit python3 -m http.server im Repo-Ordner testen und dann http://localhost:8000 öffnen — auf GitHub Pages funktioniert es ohnehin automatisch, weil dort alles über https:// ausgeliefert wird.
