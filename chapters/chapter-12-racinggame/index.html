Racing Game

Ein Top-Down-Rennspiel: lenken, beschleunigen, auf der Straße bleiben (abseits wird man langsamer), eine volle Runde über drei Wegpunkte fahren. Original: A3GPU212_RacingGame.

hitTestPoint() gegen eine Grafik → isPointInPath() gegen einen Vektorpfad

Der interessanteste Portierungs-Schritt in diesem Kapitel: Das Original prüft mit road.hitTestPoint(x, y, true) pixelgenau, ob eine Koordinate auf der (im Flash-Editor gezeichneten) Straßengrafik liegt. Canvas hat keinen Pixel-Treffertest eingebaut — der nächste Verwandte ist ctx.isPointInPath(path, x, y) gegen einen Path2D, den man sich selbst aus der Streckenform baut (hier: äußere minus innere abgerundete Fläche, per evenodd-Füllregel als "Donut").

ActionScript 3 
if (!gamesprite.road.hitTestPoint(
    carPos.x+dx+gamesprite.x,
    carPos.y+dy+gamesprite.y,
    true)) {
  if (gamesprite.side.hitTestPoint(
      ...)) {
    speed *= 1.0 - .001*timeDiff;
  } else {
    speed *= 1.0 - .005*timeDiff;
  }
}
JavaScript 
if (!ctx.isPointInPath(
    roadPath, nx, ny, "evenodd")) {
  if (ctx.isPointInPath(
      sidePath, nx, ny, "evenodd")) {
    speed *= 1.0 - .001*timeDiff;
  } else {
    speed *= 1.0 - .005*timeDiff;
  }
}

Fahrphysik: unverändert übernommen

Beschleunigung, Bremsen/Rückwärtsgang, natürliches Ausrollen ohne Tastendruck, und die Lenkung (schneller drehen bei höherer Geschwindigkeit — (speed+.1)*turnSpeed*timeDiff) sind 1:1 aus dem Original übernommen, inklusive aller Konstanten.

Zielankunft: an die eigene Streckenform angepasst

Das Original prüft die Ziellinie mit einem einfachen car.y < finish.y — das setzt eine bestimmte Streckenausrichtung voraus (senkrechte Ziellinie oben). Da unsere Strecke eine Ringform ("Donut") statt einer geraden Strecke ist, wurde checkFinishLine() auf einen Abstandstest zum Startpunkt umgestellt — dasselbe Ziel ("eine volle Runde nach allen Wegpunkten"), nur an die andere Streckenform angepasst.

↑/↓ Gas/Bremse, ←/→ lenken. Drei Wegpunkte anfahren, dann zurück zur Startlinie. Motorsound ist synthetisch erzeugt (Web Audio), da die Original- Sounds nur als Flash-Bibliothekssymbole ohne exportierte Audiodatei vorlagen.
