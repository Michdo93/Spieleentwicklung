Jigsaw Puzzle

48 Puzzleteile (8×6), zufällig verstreut. Das Interessante hier ist nicht das Ziehen selbst, sondern dass richtig zusammengesetzte Teile sich ab sofort gemeinsam bewegen. Original: A3GPU206_JigsawPuzzle.

Der eigentliche Clou: verbundene Teile-Gruppen

findLockedPieces() prüft beim Anklicken eines Teils, welche anderen Teile exakt in der richtigen relativen Position direkt daneben liegen — und zieht sie automatisch mit. Das Original löst das mit einer Liste, die nach Entfernung zum angeklickten Teil sortiert wird, und einer Schleife, die so lange neue Verbindungen sucht, bis keine mehr dazukommt. Dieser Algorithmus ist wortwörtlich derselbe in JavaScript — nur Array.sortOn() wird zu Array.sort() mit eigener Vergleichsfunktion.

sortedObjects.sortOn(
  "dist", Array.DESCENDING);

do {

  for (i=sortedObjects.length-1;
       i>=0; i--) {

} while (found);

Kein festes Ziel-Board — relative Position zählt

Auffällig: das Original prüft nicht, ob die Teile an einer bestimmten Bildschirmposition liegen, sondern nur, ob sie zueinander richtig positioniert sind (puzzleTogether() vergleicht nur Differenzen). Man kann das Puzzle also irgendwo auf der Fläche zusammensetzen — das habe ich 1:1 übernommen.

10-Pixel-Raster als Toleranz

Nach jedem Loslassen werden alle Teile auf das nächste Vielfache von 10 Pixeln gerundet (lockPiecesToGrid()). Das macht exaktes Zusammensetzen per Maus überhaupt erst praktikabel — ohne dieses Raster müsste man pixelgenau treffen.

Teil anklicken und ziehen — bereits richtig verbundene Nachbarn werden automatisch mitgezogen. Das Originalbild aus dem Buchmaterial wird unverändert verwendet.
