Racing 3D

Dieselbe Ringstrecke wie Chapter 12a (Racing Game) — diesmal aus der Ich-Perspektive statt von oben gerendert. Ein freies Fahr-Techdemo ohne festen Rundenzähler, genau wie im Original. Original: A3GPU214_Racing3D.

Eine Funktion, zwei Aufgaben

Der schönste Aspekt dieser Portierung: surfaceAt(x, z) — derselbe "abgerundetes Donut"-Test wie in Chapter 12a — treibt hier gleichzeitig die Physik (Geschwindigkeitsverlust abseits der Straße) und die Bodeneinfärbung beim Zeichnen an. Keine doppelte Pflege zweier separater Straßen-Definitionen.

Chapter 12a (von oben, Canvas-Pfad) 
if (!ctx.isPointInPath(
    roadPath, x, y, "evenodd")) {
  // abseits der Straße
}
Chapter 14c (Ich-Perspektive, reine Mathematik) 
function surfaceAt(x, z) {
  const onRoad =
    pointInRoundedRect(x,z,OUTER)
    && !pointInRoundedRect(x,z,INNER);
  return onRoad ? "road" : ...;
}

Derselbe geometrische Test, nur ohne Canvas-Pfad-API (die für Boden-Kacheln in der Ferne unpraktisch wäre) — als reine Punkt-in-abgerundetes-Rechteck-Mathematik.

Boden als projizierte Kachelraster

Statt eines vorgezeichneten Straßenbilds (wie im Original) wird der sichtbare Boden hier live als Raster aus kleinen Quadraten um die Kamera herum erzeugt, jedes einzeln über project() (Chapter 14a) in die Ansicht projiziert und nach surfaceAt() eingefärbt — Straße, Bankett oder Gelände.

Bäume: zufällig platziert, Straße gemieden

Dasselbe "würfeln, bis eine gültige Position gefunden ist"-Muster wie in mehreren früheren Kapiteln — hier mit der Bedingung "abseits der Straße" statt "nicht auf einem Hindernis".

↑ Gas, ←/→ lenken (stärker bei höherem Tempo, wie im Original). Kein Rundenzähler — freies Fahren, genau wie die Techdemo im Buch.
