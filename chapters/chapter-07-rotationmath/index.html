Rotation Math

Die Trigonometrie-Grundlagen für den Rest des Kapitels: aus einem Winkel einen Bewegungsvektor berechnen, aus einer Richtung einen Winkel. Original: A3GPU207_RotationMath (PointingArrow, MovingCar).

Zwei Richtungen derselben Formel

Beide Demos benutzen dieselbe Mathematik, nur in entgegengesetzter Richtung: Winkel → Vektor (cos/sin, für Bewegung "in Blickrichtung") und Vektor → Winkel (atan2, für "worauf zeigt dieses Objekt gerade"). AS3 und JavaScript haben hier exakt dieselbe Math-API — Math.cos, Math.sin und Math.atan2 existieren unverändert.

Der einzige Stolperstein: Grad vs. Radiant

MovieClip.rotation in AS3 ist in Grad, aber Math.cos()/Math.sin() erwarten Radiant. Das Original rechnet das an mehreren Stellen von Hand um (2*Math.PI*rotation/360). Im Web gibt es dieselbe Situation — ctx.rotate() nimmt ebenfalls Radiant. Ich habe die Umrechnung einmal als kleine Hilfsfunktion geschrieben, statt sie überall neu auszuschreiben.

ActionScript 3 (mehrfach wiederholt) 
var angle:Number =
  2*Math.PI*(car.rotation/360);
var dx:Number =
  speed*Math.cos(angle);
var dy:Number =
  speed*Math.sin(angle);
JavaScript (einmal definiert) 
function deg2rad(d) {
  return d * Math.PI / 180;
}
const angle = deg2rad(rotation);
const dx = Math.cos(angle) * speed;
const dy = Math.sin(angle) * speed;

Wo das als Nächstes gebraucht wird

  • Air Raid 2 — die Kanone dreht sich, Kugeln fliegen per cos/sin in Feuerrichtung
  • Balloon Pop — dieselbe Formel für die Kanonenkugel-Flugbahn
  • Space Rocks — Schiffsrotation, Schub in Blickrichtung, Geschoss-Richtung — alles dieselbe Formel wie hier
