3D Demos

Die Grundlagen für den Rest des Kapitels: Flash Players eingebaute "native 3D"-Eigenschaften (z, rotationX/Y/Z) und wie man ihre Wirkung ohne diese Engine von Hand nachrechnet. Original: A3GPU214_Demos (demo1–demo4).

Was Flash automatisch erledigt hat

Ab Flash 10 hatte jedes DisplayObject eingebaute 3D-Eigenschaften: z (Tiefe), rotationX, rotationY, rotationZ. Der Player berechnete Perspektive und Verdeckung automatisch — man musste nur die Werte setzen. Canvas hat dafür keine Entsprechung: Wir müssen die Perspektiven-Projektion selbst rechnen.

Die eine Formel für dieses ganze Kapitel

Das Grundprinzip der Zentralprojektion — je weiter weg (größeres z), desto kleiner und näher am Fluchtpunkt — lässt sich mit einer einzigen Formel ausdrücken. Diese Formel taucht in allen vier Unterkapiteln von Chapter 14 wieder auf.

ActionScript 3 (automatisch) 
square2.x = 275;
square2.y = 200;
square2.z = 100; // Flash Player
  // übernimmt den Rest
JavaScript (von Hand) 
function project(x, y, z) {
  const scale =
    FOCAL / (FOCAL + z);
  return {
    x: x, y: y, scale
  };
}

rotationX/Y/Z ohne echte 3D-Engine

Ohne WebGL lässt sich eine echte Flächendrehung im Raum auf einem 2D-Canvas nicht exakt nachbilden. Die Demos hier nähern das Kippen um die X-/Y-Achse als einfache Stauchung an (cos(winkel) als Höhen- bzw. Breitenfaktor) — visuell überzeugend genug, um das Prinzip zu zeigen, aber bewusst eine Vereinfachung. Rotation um die Z-Achse braucht dagegen gar keinen Trick — das ist stinknormale 2D-Rotation, die ctx.rotate() exakt kann.

Vier Tabs: reine Tiefe (z), Kippen um 30°/60°, volle 90°-Kippung (wie ein Boden), und alle drei Rotationsachsen gleichzeitig animiert.
