Word Search

Ein Buchstabenraster, Wörter per Ziehen markieren — waagerecht, senkrecht oder diagonal, auch rückwärts. Der komplette Platzierungsalgorithmus ist 1:1 aus dem Original übernommen. Original: A3GPU209_WordSearch.

Wörter ins Raster würfeln: Versuch-und-Irrtum

placeLetters() probiert bis zu 2000 Mal, ein Wort an einer zufälligen Position in eine zufällige Richtung (8 mögliche Richtungen, inklusive Diagonalen) zu platzieren. Passt es nicht (Rasterrand oder Buchstabenkonflikt), wird der nächste Versuch gestartet. Am Ende füllen zufällige Buchstaben die Lücken.

ActionScript 3 
for (var j=0; j<word.length; j++) {
  if ((x+dx*j < 0) ||
      (y+dy*j < 0) ||
      (x+dx*j >= puzzleSize) ||
      (y+dy*j >= puzzleSize)) {
    continue repeatLoop;
  }
  var cur = letters[x+dx*j][y+dy*j];
  if ((cur != "*") &&
      (cur != word.charAt(j))) {
    continue repeatLoop;
  }
}
JavaScript 
let fits = true;
for (let j=0; j<word.length; j++) {
  const x = x0+dx*j, y = y0+dy*j;
  if (x<0 || y<0 ||
      x>=puzzleSize ||
      y>=puzzleSize) {
    fits = false; break;
  }
  const cur = letters[x][y];
  if (cur !== "*" &&
      cur !== word.charAt(j)) {
    fits = false; break;
  }
}
if (!fits) continue;

AS3s continue repeatLoop; (ein benanntes continue, das direkt zur äußeren Schleife springt) gibt es in JavaScript nicht ohne Weiteres in dieser Form innerhalb verschachtelter for-Schleifen mit Label — hier reicht aber ein einfaches fits-Flag mit break und einem äußeren continue, funktional identisch.

Auswahl-Validierung: nur drei erlaubte Richtungen

isValidRange() prüft nur drei Fälle: gleiche Spalte, gleiche Reihe, oder eine exakte 45°-Diagonale (abs(dx) == abs(dy)) — alles andere wird ignoriert. Unverändert übernommen.

Rückwärts gelesen zählt auch

checkWord() vergleicht die Auswahl sowohl vorwärts als auch rückwärts gegen die Wortliste — man kann ein Wort also in beide Richtungen markieren.

Von einem Buchstaben zum letzten Buchstaben eines Worts ziehen (waagerecht, senkrecht oder diagonal) und loslassen.
