Trivia Game

Ein Multiple-Choice-Quiz aus 10 Fragen, geladen aus externen Daten, Antworten gemischt, laufender Punktestand. Original: A3GPU210_TriviaGame.

Canvas oder DOM? Hier eindeutig DOM.

Alle bisherigen Spiele in diesem Repo liefen über <canvas>, weil sie Bewegung, Kollision oder freie Zeichnung brauchten. Ein Quiz ist im Kern ein Formular: Text, Buttons, Listen. Dafür ist echtes HTML die bessere Wahl — Textumbruch, Barrierefreiheit und Styling funktionieren "geschenkt", ganz ohne selbst geschriebene Layout-Logik. AS3s createText()-Hilfsfunktion (die für jedes Textfeld manuell Format, Zeilenumbruch und Größe setzt) entfällt komplett — das übernimmt CSS.

Antworten mischen: derselbe Algorithmus

ActionScript 3 
function shuffleAnswers(
    answers:XMLList) {
  var shuffled = [];
  while (answers.child("*")
      .length() > 0) {
    var r = Math.floor(
      Math.random()*answers
      .child("*").length());
    shuffled.push(
      answers.answer[r]);
    delete answers.answer[r];
  }
  return shuffled;
}
JavaScript 
function shuffleAnswers(
    originalAnswers) {
  const pool = [...originalAnswers];
  const shuffled = [];
  while (pool.length > 0) {
    const r = Math.floor(
      Math.random()*pool.length);
    shuffled.push(pool[r]);
    pool.splice(r, 1);
  }
  return shuffled;
}

Erste Antwort in den Originaldaten ist immer die richtige (correctAnswer = item.answers[0]) — die Mischfunktion zieht zufällig Elemente heraus, bis alle in neuer Reihenfolge in einem neuen Array stehen. Exakt dasselbe Prinzip in beiden Sprachen.

Ablauf bleibt 1:1

Frage anzeigen → Antwort anklicken → richtige/falsche Antwort markieren, alle anderen ausblenden → "Continue"-Button → nächste Frage, bis alle 10 durch sind. Genau der Ablauf aus pressedGameButton()/finishQuestion() im Original.
