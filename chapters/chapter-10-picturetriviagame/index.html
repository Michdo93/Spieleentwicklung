Picture Trivia Game

Ein Quiz, bei dem Frage oder Antworten auch Bilder statt Text sein können — gemischt in derselben Datenstruktur. Original: A3GPU210_PictureTriviaGame.

Gemischte Typen: type="text" vs. type="file"

Der Clou im Original: jede Frage und jede Antwort trägt ein type-Attribut. Ist es "text", wird ein Textfeld erzeugt; ist es "file", lädt ein Loader stattdessen eine externe SWF-Bilddatei nach. Dieselbe Fallunterscheidung steckt hier im JS-Code — nur dass aus "file" "image" wurde und statt eines Loader-Objekts direkt Inline-SVG eingesetzt wird.

ActionScript 3 
if (question.@type == "text") {
  questionField = createText(
    question, ...);
} else {
  var loader = new Loader();
  loader.load(new URLRequest(
    "triviaimages/"+question));
  questionSprite.addChild(loader);
}
JavaScript 
qEl.innerHTML =
  item.question.type === "text"
    ? item.question.value
    : imageHTML(item.question.value);

Bilder: selbst gezeichnet statt Originaldateien

Die Original-Bilder lagen als kompilierte .swf-Dateien vor (equilateral.swf, right.swf, isosceles.swf, scalene.swf, italy.swf) — ein Format, das kein Browser mehr abspielen kann und aus dem sich keine Bilddaten extrahieren lassen. Da für dieses Kapitel also wirklich keine verwendbare Originaldatei vorlag, habe ich fünf kleine Ersatzgrafiken selbst als Inline-SVG gezeichnet: vier klar unterscheidbare Dreiecksarten und eine stark vereinfachte, stilisierte Silhouette Italiens.

Kein Server nötig

Anders als die beiden vorigen Trivia-Kapitel lädt dieses hier keine externe JSON-Datei — die drei Fragen stecken direkt im Skript. Das spart den lokalen Server, den fetch()-basierte Seiten sonst brauchen (siehe Chapter 10a).

Die Antwortstruktur bleibt identisch zu Chapter 10b/10c — nur womit eine Antwort dargestellt wird (Text oder Bild), ändert sich pro Frage.
