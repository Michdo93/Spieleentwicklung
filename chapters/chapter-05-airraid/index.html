Air Raid

Das erste echte Spiel im Buch: eine Flak-Kanone schießt auf Flugzeuge. Hier lernt man die Grundmuster, die praktisch jedes Arcade-Spiel braucht — eigene Klasse pro Spielobjekt, zeitbasierte Bewegung statt Frame-Zählung, Kollisionserkennung, Objektlisten verwalten. Original: A3GPU205_AirRaid.

Architektur: vier Klassen, eine Aufgabe pro Klasse

Das Original teilt das Spiel sauber auf: AirRaid (Spielleitung/Zustand), AAGun (Spielerfigur), Airplane (Gegner) und Bullet (Projektil). Diese Aufteilung habe ich 1:1 übernommen — nur dass aus AS3-Klassen JS-Klassen werden und aus MovieClip-Vererbung ein einfaches this.game-Referenz-Muster, weil es im Canvas keine automatische Anzeigeliste gibt, die "weiß", wer zu wem gehört.

ENTER_FRAME → requestAnimationFrame + Delta-Zeit

Das Original berechnet Bewegung nicht in "Pixel pro Frame", sondern in "Pixel pro Sekunde" über flash.utils.getTimer() — dadurch bleibt das Spiel gleich schnell, egal wie die Bildrate schwankt. Das Web-Äquivalent ist exakt dasselbe Prinzip mit requestAnimationFrame und der Zeitdifferenz zwischen zwei Frames.

ActionScript 3 
addEventListener(
  Event.ENTER_FRAME, moveGun
);

function moveGun(e:Event) {
  var dt:int =
    getTimer() - lastTime;
  lastTime += dt;
  x += speed * dt / 1000;
}
JavaScript 
requestAnimationFrame(loop);

function loop(now) {
  const dt =
    (now - lastTime) / 1000;
  lastTime = now;
  x += speed * dt;
  requestAnimationFrame(loop);
}

hitTestObject() → eigene Rechteck-Kollision

Flash brachte hitTestObject() als fertige Bounding-Box-Kollision mit. Im Canvas gibt es das nicht eingebaut — wir schreiben die klassische Achsen-parallele Rechteck-Überlappung (AABB) selbst, sie ist nur wenige Zeilen lang und wird im restlichen Buch noch oft gebraucht.

ActionScript 3 
if (bullet.hitTestObject(plane)) {
  plane.planeHit();
  bullet.deleteBullet();
}
JavaScript 
function rectsHit(a, b) {
  return a.x < b.x+b.w && a.x+a.w > b.x
      && a.y < b.y+b.h && a.y+a.h > b.y;
}

Objektverwaltung: Arrays statt Anzeigeliste

airplanes und bullets sind im Original einfache Arrays, über die rückwärts iteriert wird, wenn währenddessen Elemente entfernt werden (klassisches Muster, um Indexverschiebung beim Löschen zu vermeiden). Das ist in JavaScript identisch übernommen — splice() statt Array.splice() gibt es hier wortwörtlich in beiden Sprachen.

Grafik: Vektorzeichnung statt Bibliothekssymbole

Kanone, Flugzeuge und Projektile waren im Original Flash-Bibliothekssymbole (in der FLA gezeichnet, per Class-Verknüpfung an die AS3-Klassen gebunden). Da es diese Bibliothek im Web nicht gibt, zeichne ich die Formen direkt mit der Canvas-2D-API — funktional gleichwertig zu graphics.beginFill()/drawRect(), nur ohne den Umweg über den Flash-Editor.

Steuerung: ← / → bewegen die Kanone, Leertaste feuert. 20 Schuss, keine Nachladung — genau wie im Original.

Bereit? 20 Schuss, so viele Flugzeuge wie möglich treffen.
