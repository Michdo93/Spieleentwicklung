Memory Game

Ein "Simon"-Klon: Der Computer spielt eine wachsende Ton-/Licht-Sequenz vor, der Spieler wiederholt sie. Ein Fehler beendet das Spiel. Original: A3GPU204_MemoryGame.

Verketteter Timer statt Schleife

Interessant an MemoryGame.as ist, wie die Sequenz abgespielt wird: nicht mit einer for-Schleife (die würde alle Lichter gleichzeitig zeigen), sondern mit einem Timer, der pro Tick einen Schritt weiterspringt. Das JS-Äquivalent ist eine Kette von setTimeout()-Aufrufen, bei der jeder Aufruf den nächsten selbst plant — dasselbe Grundprinzip.

ActionScript 3 
lightTimer = new Timer(
  1000, playOrder.length+1);
lightTimer.addEventListener(
  TimerEvent.TIMER,
  lightSequence);
lightTimer.start();

function lightSequence(e) {
  var step =
    e.currentTarget
     .currentCount-1;
  if (step < playOrder.length) {
    lightOn(playOrder[step]);
  } else {
    startPlayerRepeat();
  }
}
JavaScript 
playSequence(step) {
  if (step >= this.playOrder
      .length) {
    this.startPlayerRepeat();
    return;
  }
  this.lightOn(
    this.playOrder[step]);
  setTimeout(
    () => this.playSequence(
      step + 1),
    700
  );
}

Sequenz merken: ein einfacher Array-Vergleich

Die eigentliche Spiellogik ist erfreulich simpel: repeatOrder ist eine Kopie der bisherigen Sequenz, bei jedem Klick wird das erste Element mit shift() entnommen und verglichen. Stimmt es nicht überein, ist das Spiel vorbei — dieselbe Methode existiert unverändert in JavaScript-Arrays.

Sounds: fünf Töne, ein Sample pro Licht

Die fünf note1.mp3 … note5.mp3-Dateien aus dem Originalmaterial werden unverändert übernommen — jedes Licht bekommt seinen eigenen Ton, genau wie im Buch.

Steuerung: einfach auf die Kreise klicken, während "Jetzt du" angezeigt wird. Die Sequenz wird bei jeder erfolgreichen Runde um ein Element länger.

Zuschauen, zuhören, wiederholen. Wie lang wird deine Sequenz?
