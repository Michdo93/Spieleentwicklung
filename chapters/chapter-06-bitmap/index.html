Bitmap

Der Grundbaustein für die beiden Puzzles in diesem Kapitel: ein Bild laden und in ein Raster aus Einzelteilen zerschneiden. Original: A3GPU206_Bitmap.

Loader/BitmapData → Image/drawImage

AS3 lädt externe Bilder asynchron mit Loader + URLRequest und schneidet Ausschnitte mit BitmapData.copyPixels() heraus. Im Browser übernimmt Image das Laden (mit dem load-Event statt Event.COMPLETE), und drawImage() mit den neun-Parameter-Overload kann direkt einen Ausschnitt aus der Quelle in ein Zielrechteck zeichnen — eine Funktion statt zwei Schritte.

ActionScript 3 
var loader:Loader = new Loader();
loader.contentLoaderInfo
  .addEventListener(
    Event.COMPLETE, done);
loader.load(
  new URLRequest("bild.jpg"));

function done(e:Event) {
  var img:Bitmap =
    Bitmap(e.target.loader
      .content);
  var piece = new BitmapData(
    pw, ph);
  piece.copyPixels(
    img.bitmapData,
    new Rectangle(x*pw,y*ph,pw,ph),
    new Point(0,0));
}
JavaScript 
const img = new Image();
img.src = "bild.jpg";
img.onload = () => {
  ctx.drawImage(
    img,
    x*pw, y*ph, pw, ph,  // Quelle
    dx, dy, pw, ph        // Ziel
  );
};

Warum das ein eigenes Mini-Kapitel ist

Sowohl das Jigsaw- als auch das Sliding-Puzzle bauen direkt auf diesem Muster auf — ein Bild in numPiecesHoriz × numPiecesVert Rechtecke zerlegen, jedes Teil als eigenständiges Objekt mit Position behandeln. Wer dieses Beispiel versteht, versteht damit auch schon die Kernidee beider folgenden Spiele.

Das Originalbild aus dem Buchmaterial (testimage.jpg) wird unverändert verwendet — hier reicht ein einzelnes Standbild, kein generiertes Ersatzbild nötig.
